# 复式记账改造计划

## 核心变更

**旧（单式）→ 新（复式）：**
```
一条记录 = { source, amount, category }
                ↓
一笔交易 = { description, timestamp, entries: [
  { account: "现金", debit: 35 },
  { account: "餐饮费用", credit: 35 }
]}
```

## 数据结构

**账户 (Account)** — 有余额的概念
- 资产类：现金、银行卡、微信钱包、支付宝
- 收入类：工资收入、投资收益
- 费用类：餐饮、交通、购物、居住、医疗、娱乐、教育
- 负债类：信用卡（可选）

**交易 (Transaction)** — 至少一借一贷
- 收入：借→资产，贷→收入
- 支出：借→费用，贷→资产
- 转账：借→资产A，贷→资产B
